    Introdução

    Bem-vindo(a) a API do Brasil Bookings!

    Autenticação

    Para autenticação utilize o seguinte código:

    curl -X POST https://app.brbookings.com/api/CAMINHODOOBJETO \
-d 'api_key=MINHAAPIKEY' \

    Você deve substituir MINHAAPIKEY por sua API Key, e deve substituir CAMINHODOOBJETO pelo objeto que você deseja acessar.

    A API do Brasil Bookings utiliza API Keys para autorizar o acesso à API. A API Key deve ser informada em todos os requests ao servidor, no header da requisição, conforme o exemplo a seguir:

    X-API-KEY: MINHAAPIKEY

    Você deve substituir MINHAAPIKEY com a sua API key exclusiva.

    Tipos de Quarto

    Através da rota /rooms_api/rooms e suas derivadas, você pode retornar os dados básicos dos tipos de quartos existentes.

    Objeto rooms_api/rooms

    GET https://app.brbookings.com/api/rooms_api/rooms

    RESPOSTA (exemplo)

    {
  "RoomsRS": {
    "0": "Rooms",
    "Rooms": {
      "Success": "",
      "HotelCode": "1",
      "0": {
        "ID": "1",
        "Title": "Quarto 1",
        "Adults": "3",
        "Children": "2"
      },
      "1": {
        "ID": "2",
        "Title": "Quarto 2",
        "Adults": "2",
        "Children": "0"
      },
      "2": {
        "ID": "3",
        "Title": "Quarto 3",
        "Adults": "4",
        "Children": "0"
      },
      "3": {
        "ID": "4",
        "Title": "Quarto 4",
        "Adults": "5",

    integer    		 Quantidade máxima de hóspedes adultos do quarto
    Children
    integer    		 Quantidade máxima de hóspedes crianças do quarto

    "AvailStatusMessages": {

            "InvCode": "1"

        },
        {
          "BookingLimit": "14",
          "StatusApplicationControl": {
            "Start": "2016-11-21",
            "End": "2016-11-22",
            "InvCode": "1"

        {
          "BookingLimit": "14",
          "StatusApplicationControl": {
            "Start": "2016-11-22",

    Atualizar Restrições e Estadia Mínima

    Através da rota /restrictions_api/update_restrictions, você pode atualizar as restrições e estadias mínimas dos quartos por período e tipo de tarifa.

    Atualizando restrições e estadias mínimas

    POST https://app.brbookings.com/api/restrictions_api/update_restrictions (exemplo)

    {
  "RestrictionsRQ": {
    "AvailStatusMessages": {
      "HotelCode": "1",
      "AvailStatusMessage": [
        {
          "StatusApplicationControl": {
            "Start": "2016-11-02",
            "End": "2016-11-03",
            "RatePlanCode": "1",
            "InvCode": "1"
          },
          "LengthsOfStay": {
            "LengthOfStay": {
              "Time": "3",
              "TimeUnit": "Day",
              "MinMaxMessageType": "MinLOS"
            }
          }
        },
        {
          "StatusApplicationControl": {
            "Start": "2016-11-02",
            "End": "2016-11-03",
            "RatePlanCode": "1",
            "InvCode": "1"
          },
          "RestrictionStatus": [{
              "Restriction": "Arrival",
              "Status": "Open"
            },
            {
              "Restriction": "Departure",
              "Status": "Close"

    }
  }
}

    RESPOSTA (exemplo)

    "Success": ""
  }
}

    Start

    End
    string    		 Data final do período
    Time
    integer    		 Quantidade de dias que deve se aplicar estadia mínima
    TimeUnit
    string    		 Tipo de unidade de tempo
    Valores possíveis: Day
    MinMaxMessageType
    string    		 Tipo de restrição

    string    		 Tipo de bloqueio
    Valores possíveis: Arrival, Departure

    string    		 Status da restrição

    Confirmação da baixa de reservas

    Após baixar uma reserva, você deve enviar um POST confirmando que efetuou a baixa da mesma em nosso sistema.

    POST https://app.brbookings.com/api/reservations_api/reservations (exemplo)

    {
  "ReservationRQ": {
    "Reservation": [
      {
        "HotelCode": "1",
        "ReserveId": "bo415874340"
      },
      {
        "HotelCode": "1",
        "ReserveId": "14504642041"
      }
    ]
  }
}

    RESPOSTA (exemplo)

    {
  "ReservationRS": {
    "Reservations": [
      {
        "ReserveId": "bo415874340",
        "HotelCode": "1",
        "Success": ""
      },
      {
        "ReserveId": "14504642041",
        "HotelCode": "1",
        "Success": ""
      }
    ]
  }
}
    Propriedade Descrição
    HotelCode
    string    		 Código do hotel no Brasil Bookings
    ReserveId
    integer    		 ID do da reserva
    Success
    string    		 Indica que a baixa da reserva foi concluída com sucesso

    Erros

    Código Significado
    BBEA1 Há X reserva(s) pendentes de sincronização. Você deve sincronizar todas as reservas pendentes antes de atualizar a disponibilidade dos quartos.
    BBEA2 Nenhuma disponibilidade pôde ser atualizada.
    BBEA3 A disponibilidade não pôde ser atualizada. A mensagem varia de acordo com o que impediu a atualização de ser realizada.
    BBEA4 Quantidade enviada é inválida, ou está faltando.
    BBER1 Valores por pessoa por noite não enviados na requisição.
    BBER2 Nenhuma tarifa pôde ser atualizada.
    BBER3 O valor da tarifa não foi especificado.
    BBER4 O número de hóspedes para a tarifa não foi especificado.
    BBER5 A tarifa não pôde ser atualizada. A mensagem varia de acordo com o que impediu a atualização de ser realizada.
    BBERE1 Nenhuma restrição pôde ser atualizada.
    BBERE2 Dados da restrição inválidos, ou não enviados.
    BBERE3 A restrição não pôde ser atualizada. A mensagem varia de acordo com o que impediu a atualização de ser realizada.
    BBERE4 Estadia mínima ou bloqueio de checkin/checkout não enviados.
    BBEBR1 Nenhuma disponibilidade pôde ser atualizada.
    BBEBR2 O bloqueio/desbloqueio não pôde ser feito. A mensagem varia de acordo com o que impediu a atualização de ser realizada.
    BBEBR3 Status da Tarifa enviado é inválida, ou está faltando.
    BBERO1 Nenhum item encontrado
    BBERO2 Nenhum quarto retornado. A mensagem varia de acordo com o que impediu a atualização de ser realizada.
    BBERA1 Nenhum item encontrado.
    BBERA2 Nenhum tipo de tarifa encontrado. A mensagem varia de acordo com o que impediu a atualização de ser realizada.
    BBERES1 Nenhuma reserva encontrada.

    BBERES3 Nenhuma reserva pôde ser confirmada.

    BBE1 A data inicial deve ser menor que a final, ou está faltando.

    BBE3 Data inicial inválida, ou está faltando.
    BBE4 ID do quarto é inválido, ou está faltando.
    BBE5 ID do tipo de tarifa não enviado na requisição.
